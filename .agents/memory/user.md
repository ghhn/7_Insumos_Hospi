# Identidad y Preferencias del Usuario (Contexto Persistente)

## Organización
- **Nombre del Proyecto**: 7_Insumos_Hospi
- **Dominio**: Gestión de presupuestos, metrados y adquisición de insumos.
- **Roles**: Arquitectura de Supabase, análisis de compatibilidad de insumos, reportes en Streamlit.

## Preferencias Técnicas
- **Base de Datos**: PostgreSQL / Supabase
  - Regla estricta: Siempre referenciar la guía maestra `SQL_Architecture_Master_Guide.md` al realizar cambios estructurales (aunque los cambios de esta etapa son de solo migración).
- **Control de Versiones**:
  - Regla estricta: Cuando se trabaje en el proyecto del mismo nombre que la carpeta local, lo primero que se debe hacer es un commit en git local (`git add .`, `git commit`).
- **Autonomía y Skills**:
  - Preferencia: Usar siempre los skills locales primero (`.agents/skills/`).
  - Libertad absoluta: Si no hay un skill adecuado, la IA debe pensar, investigar y desarrollar la mejor solución posible de manera autónoma.

## Formato de Diseño / Frontend
- Preferencias (Skills): El usuario posee skills avanzados (e.g. `exportador_reportes`, `design-taste-frontend`, `high-end-visual-design`) que denotan una expectativa de interfaces premium y formatos altamente profesionales.
